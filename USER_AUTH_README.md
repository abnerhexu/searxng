# SearXNG 用户认证功能使用说明

## 新增功能

本次开发为 SearXNG 添加了以下功能：

1. **用户注册和登录系统**
2. **搜索历史记录**（查看和删除）
3. **用户设置持久化**（分用户隔离）
4. **管理员用户管理**

## 安装依赖

```bash
pip install -r requirements.txt
```

## 初始化数据库和创建管理员

首次使用需要创建管理员账户：

```bash
python create_admin.py
```

按提示输入管理员用户名、邮箱和密码。

## 功能说明

### 1. 用户注册和登录

- 访问 `/account/register` 注册新用户
- 访问 `/account/login` 登录
- 登录后可以在导航栏看到用户名和登出按钮

### 2. 搜索历史

- 登录用户的每次搜索会自动记录
- 访问 `/account/history` 查看搜索历史
- 可以删除单条历史或清空所有历史

### 3. 用户设置持久化

- 登录用户的设置会保存到数据库
- 设置跨设备同步
- 未登录用户仍使用 Cookie 存储设置

### 4. 管理员功能

管理员可以访问 `/admin/users` 进行用户管理：

- 查看所有用户列表
- 启用/禁用用户账户
- 设置/取消管理员权限

## 数据库位置

用户数据库文件默认位于：`./data/users.db`（相对于项目根目录）

如果在 `settings.yml` 中配置了 `general.data_dir`，则使用该路径。

配置示例（在 `settings.yml` 中添加）：

```yaml
general:
  data_dir: /var/lib/searxng/data  # 自定义数据目录
```

如果不配置，默认使用项目根目录下的 `data` 文件夹。

## 向后兼容

- 未登录用户可以正常使用搜索功能
- 未登录用户的设置仍保存在 Cookie 中
- 不影响现有的搜索引擎配置

## 技术实现

- **认证框架**: Flask-Login
- **数据库**: SQLAlchemy + SQLite
- **密码加密**: Werkzeug Security
- **会话管理**: Flask Session

## 注意事项

1. 首次运行会自动创建数据库表
2. 建议定期备份 `users.db` 文件
3. 搜索历史仅对登录用户记录
4. 管理员不能删除自己的管理员权限
